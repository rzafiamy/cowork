Agentic Document Workflow

A visual deep-dive into how an AI agent autonomously creates a complete, coherent document from a single user request.

1. Plan

The agent deconstructs the user's request and creates a structured outline (JSON). This blueprint ensures the document is logically organized before any content is written.

2. Draft

Following the plan, the agent writes the document section by section. It uses a "rolling context window"—a summary of early sections and the full text of the last section—to maintain coherence and flow.

3. Review

A "reviewer" persona analyzes the complete first draft, specifically looking for redundancy, repetition, and logical gaps. It generates a list of actionable feedback.

4. Refine

The final agent persona takes the original draft and the reviewer's feedback, then performs a final polish. It incorporates the suggestions, adds a strong intro/conclusion, and ensures perfect formatting.

The Memory Bank

The Agent's Short-Term Workspace

Since each LLM call is stateless, the agent's code uses `localStorage` as a temporary "notebook" to maintain context. This single, consolidated entry is created at the start of the task and **completely deleted** upon completion.

// localStorage key: memory_session_xyz

{

"user_requirements": "...",

"outline": [...],

"draft": "## Title...",

"reviewer_feedback": "..."

}

Analyzing the Trace

Identifying Tool Calls

The agent's trace log reveals its thought process. A **Tool Call** is easily identified when the `result` object contains a `tool_calls` array instead of a simple text `content` string. This indicates the LLM was forced to output structured data.

{

"step": "Creating document outline",

"result": {

"tool_calls": [

{ "function": { "name": "set_document_plan", ... } }

]

}

}
